# 硬件测试记录系统 — 使用手册

## 系统简介

本系统用于硬件设备的批量测试管理，覆盖批次创建 → 设备测试 → 故障处置 → 返厂跟踪 → 入库完结的完整生命周期。

**设备状态流转说明**：

| 阶段 | 状态流转 | 触发条件 |
|------|---------|---------|
| 正常设备 | 待测试 → 正常 | 测试通过 |
| 故障上报 | 待测试 → 故障（待处理） | 测试不通过 |
| 返厂发起 | 待处理 → 待返厂 | 人工标记待返厂 |
| 返厂发出 | 待返厂 → 返厂中 | 返厂跟踪创建订单并关联设备 |
| 返厂回厂 | 返厂中 → 已返厂 | 返厂单状态改为"完成" |
| 重新入库 | 已返厂 → 已入库 | 人工标记入库 |
| 直接入库 | 待返厂 → 已入库 | 跳过返厂流程直接入库 |

## 仪表盘

系统首页看板，展示全局数据概览。

### 主要功能
- **实时统计卡片**：故障率、平均返厂周期、返厂设备总数、入库数量
- **状态分布环图**：设备状态（正常/故障/待测试）和处置状态（待返厂/返厂中/已返厂/待处理/已入库）双环分布
- **测试趋势图**：按时间范围（7天/30天/90天/1年）查看测试数量趋势
- **批次概览表**：各批次良品率、设备数、完成进度一览

### 使用说明
1. 登录后默认进入仪表盘，数据每 30 秒自动刷新
2. 点击统计卡片可跳转对应列表页面
3. 环图悬浮显示具体数量，支持点击图例高亮

## 批次管理

对批量到货的硬件设备按批次进行统一管理。

### 主要功能
- **新建批次**：填写批次号、描述、测试标准、预计完成时间
- **批次列表**：查看所有批次的状态、设备数、故障数、良品率
- **批次操作**：启动、暂停、完成、取消批次；编辑批次信息
- **批次详情**：点击批次号进入，查看批次下所有设备测试情况
- **删除批次**：管理员权限，删除批次及其下所有设备（需二次确认）
- **导出报表**：支持导出单个批次的 Excel 测试数据

### 批次详情页

进入批次详情后依次展示：

1. **批次信息卡片**：批次号、状态、创建时间、测试员（所有参与测试的人员汇总）、设备统计
2. **分段进度条**：绿/红/黄分别对应正常/故障/待测设备，悬浮显示具体台数及占比
3. **KPI 指标面板**：良品率、故障率、测试完成率
4. **设备列表**：该批次下所有设备，支持搜索、筛选、多选批量导出和批量修改处置

### 状态说明
| 状态 | 含义 |
|------|------|
| 计划中 | 批次已创建，尚未启动测试 |
| 进行中 | 测试正在进行中 |
| 已暂停 | 测试暂时中断 |
| 已完成 | 该批次所有设备测试完毕（自动填入实际完成时间） |
| 已取消 | 批次被取消 |

### 使用说明
1. 点击「新建批次」创建批次，关联测试标准
2. 批次创建后默认"计划中"，点击「启动」转为"进行中"
3. 在批次详情内导入或添加设备，开始测试
4. 所有设备测试完毕且处置完结后，点击「完成批次」

## 设备列表

管理所有硬件设备的测试记录。

### 主要功能
- **设备列表**：分页查看所有设备，支持按 MAC、IP、单号、操作员搜索
- **多维度筛选**：按状态（全部/正常/故障/待测试）、批次、处置方式（全部/待处理/待返厂/返厂中/已返厂/已入库）筛选
- **排序**：支持按设备编号、测试时间、批次号排序
- **新增测试**：手动录入单个设备的测试信息
- **批量导入**：在批次详情内批量导入设备（未填状态默认"待测试"）
- **编辑/删除**：修改测试信息或删除记录（管理员权限）
- **批量修改**：勾选多台设备后批量修改处置方式
- **导出**：导出筛选后的设备列表为 Excel

### 显示字段
| 字段 | 说明 |
|------|------|
| 设备编号 | MAC 地址去冒号大写，自动生成 |
| 板卡 MAC | 默认隐藏列，仍可通过搜索框搜索 |
| 无线 MAC | 无线网卡 MAC 地址 |
| IP 地址 | 设备分配 IP |
| 测试状态 | 正常 / 故障 / 待测试 |
| 处置状态 | 待处理 / 待返厂 / 返厂中 / 已返厂 / 已入库 |
| 所属批次 | 关联的批次号 |
| 测试时间 | 最近一次测试时间 |
| 操作员 | 测试执行人员 |
| 备注 | 测试备注信息 |

### 编辑设备弹窗
- 设备有关联批次时，自动查询该批次测试标准并展示对应指标
- 无批次或批次未设置标准时不显示指标区域
- 测试结果"故障"时自动弹出故障原因填写框

## 故障管理

集中管理所有故障设备的处置流程。

### 主要功能
- **故障列表**：展示所有状态为"故障"的设备，含设备编号列
- **故障原因**：记录故障现象和原因描述
- **处置方式**：标记处置状态，支持多选筛选
- **批量修改**：勾选多台设备后批量修改处置方式
- **导出**：导出故障设备 Excel 列表

### 处置类型与流转

| 处置 | 说明 | 后续操作 |
|------|------|---------|
| 待处理 | 故障已记录，尚未处置 | 人工分析后决定处置方式 |
| 待返厂 | 标记为待返厂，但未关联返厂单 | 前往返厂跟踪页面创建返厂单 |
| 返厂中 | 已关联返厂单，设备在返厂途中 | 等待返厂单状态变更为"完成"后自动变"已返厂" |
| 已返厂 | 返厂完成，设备已回到仓库 | 可重新入库 |
| 已入库 | 设备最终入库 | 整个流程完结 |

> 返厂中 → 已返厂 的状态变更由系统自动完成：当返厂跟踪中的订单标记为"完成"时，关联设备自动变为"已返厂"。

## 返厂跟踪

对返厂维修设备进行全流程跟踪管理。

### 主要功能
- **新建返厂记录**：为"待返厂"设备创建返厂跟踪单，自动生成返厂单号
- **关联设备**：选择故障设备关联到返厂单（仅可选"待返厂"状态设备）
- **状态管理**：进行中 / 完成 / 取消
- **快递追踪**：记录快递公司和快递单号，支持快递 100 自动查询
- **设备联动**：返厂单完成 → 设备自动变"已返厂"；返厂单取消/删除 → 设备恢复"待返厂"
- **导出**：导出返厂跟踪数据 Excel

### 状态联动规则

| 操作 | 设备状态变化 |
|------|------------|
| 创单 + 关联设备 | 待返厂 → 返厂中 |
| 编辑关联设备（新增） | 待返厂 → 返厂中 |
| 编辑关联设备（移除） | 返厂中 → 待返厂 |
| 返厂单状态 → 完成 | 返厂中 → 已返厂 |
| 返厂单状态 → 取消 | 返厂中 → 待返厂 |
| 删除返厂单 | 返厂中/已返厂 → 待返厂 |

### 使用说明
1. 故障设备标记为「待返厂」后，前往返厂跟踪页面
2. 点击「新建返厂记录」，填写快递信息并选择关联设备
3. 设备维修回来后，编辑返厂记录将状态改为「完成」
4. 系统自动将关联设备的处置状态更新为「已返厂」

## 标准管理

定义和管理硬件测试的评定标准，用于设备测试时的自动判定。

### 主要功能
- **新建标准**：定义各测试指标的合格范围和加权比重
- **标准列表**：查看所有测试标准
- **编辑/删除**：修改或删除标准
- **批次关联**：在创建批次时关联测试标准

### 标准指标示例
| 指标 | 合格阈值 | 权重 | 说明 |
|------|---------|------|------|
| 吞吐量 | ≥ 800 Mbps | 30% | 网络吞吐量测试 |
| 丢包率 | ≤ 0.1% | 25% | 数据包丢失率 |
| 延迟 | ≤ 5 ms | 20% | 平均网络延迟 |
| 误码率 | ≤ 1e-6 | 25% | 数据传输误码率 |

### 使用说明
1. 创建测试标准，定义指标阈值和权重
2. 创建批次时关联对应标准
3. 设备测试时自动按标准判定是否合格

## 常见问题

**Q: 设备状态和处置状态有什么区别？**
A: 设备状态（status）表示测试结果（正常/故障/待测试），处置状态（fault_disposition）表示故障设备的处理阶段（待处理→待返厂→返厂中→已返厂→已入库）。正常设备无处置状态。

**Q: 设备列表搜索输入后无结果？**
A: 确认搜索关键词正确。支持 MAC 地址、IP 地址、单号、操作员姓名。板卡 MAC 列虽隐藏但仍可搜索。

**Q: 如何批量导入设备？**
A: 进入批次详情页，点击「添加设备」，按提示格式批量粘贴设备信息。未填写状态时默认设为「待测试」。

**Q: 故障设备如何走完返厂流程？**
A: ① 故障管理页将处置改为「待返厂」 → ② 返厂跟踪页新建返厂单并关联设备（自动变「返厂中」） → ③ 返厂单标记「完成」（设备自动变「已返厂」） → ④ 设备处置改为「已入库」完成闭环。

**Q: 返厂单完成后设备状态没有自动更新？**
A: 系统在返厂单状态改为「完成」时会自动将关联设备的处置变为「已返厂」。若未生效，请检查设备是否仍关联在该返厂单下。

**Q: 批次进度条的颜色代表什么？**
A: 绿色 = 正常设备，红色 = 故障设备，黄色 = 待测设备。鼠标悬浮可查看各段具体台数和占比。

**Q: 常用命令的置顶功能如何使用？**
A: 鼠标悬停命令卡片，点击图钉图标即可置顶。置顶命令始终显示在列表最上方，每位测试员的置顶互不影响。

**Q: 如何配置快递单号自动查询？**
A: 前往系统配置页填写快递100的 customer 和 key，保存后在返厂跟踪页面点击快递单号即可自动查询物流信息。

**Q: 删除返厂记录后设备状态会恢复吗？**
A: 会。删除返厂单时，系统自动将关联设备的处置恢复为「待返厂」，并清空设备的快递公司和返厂单号。
